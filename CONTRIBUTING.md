# Contributing to Decomposer

The Decomposer is a cognitive protocol, not a software library. Contributions are welcome — but they must be *honest* contributions.

## What We Accept

- **Protocol improvements**: Better definitions, edge cases, sharper formulations
- **New adapters**: Platform-specific rendering rules (e.g., Slack, Notion, LinkedIn)
- **Translation**: Localized versions of the protocol
- **Application examples**: Real-world case studies of using the protocol
- **Bug fixes**: Errors, inconsistencies, or ambiguities in the documentation

## What We Don't Accept

- **Feature requests disguised as "modules"**: The protocol is complete. It does not need "a user management module" or "a payment integration."
- **Jargon**: If your contribution uses phrases like "leverage best practices" or "synergize core competencies," it will be rejected.
- **Unfalsifiable contributions**: Every claim in your contribution must have a testable survival condition.

## How to Contribute

1. Fork the repository
2. Create a branch: `git checkout -b feat/your-change`
3. Make your changes
4. Run the Anti-Shell Self-Check against your contribution:
   - Does every node have a survival condition?
   - Is there any jargon?
   - Is there an explicit "I don't know" where appropriate?
   - Can the user name the first step in one sentence?
5. Submit a pull request with a clear description of what changed and why

## The Ultimate Test

If you cannot explain why your contribution matters in one sentence, it probably doesn't.
